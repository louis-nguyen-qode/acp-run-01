# .context — Living Architecture Documentation

This directory is the shared knowledge base for all agents working on this repo. It is updated alongside code changes, in the same PR, gated by the same CI.

## Structure

```
.context/
├── README.md               ← you are here
├── index.json              ← generated by orchestrator (do not edit by hand)
├── linear-map.json         ← Linear issue ↔ context node mapping
├── architecture/
│   ├── overview.md         ← system overview: components, data flow
│   ├── decisions/          ← Architecture Decision Records (ADRs)
│   │   └── README.md       ← ADR index and conventions
│   └── contracts/          ← Interface contracts between modules
│       └── README.md
├── primitives/             ← Reusable building blocks (hooks, utilities, components)
│   └── README.md
└── conventions/            ← Patterns this codebase uses
    └── README.md
```

## How workers use this

1. Read `architecture/overview.md` first for system context.
2. Run `context-tree query "<topic>"` to find relevant nodes.
3. Read `architecture/contracts/` for any module your task touches.
4. Read `primitives/` so you don't rebuild what exists.
5. Update `.context/` in the same PR when you add a primitive, change a contract, or establish a new convention.

## Frontmatter format

Every `.context/` markdown file must have this frontmatter:

```yaml
---
title: "Short title"
type: "overview" | "decision" | "contract" | "primitive" | "convention"
status: "active" | "deprecated" | "proposed"
updated: "YYYY-MM-DD"
linear: "QOD-123"   # optional — the issue that created this node
---
```

CI (`scripts/check-frontmatter.ts`) validates this format.
